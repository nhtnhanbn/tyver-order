Format description

Your configuration includes your input data - that is, your candidates and the groups (also known as parties or tickets) that they belong to - as well as properties relating to the ballot layout and formality requirements.

A configuration is a JSON string that has the following properties:

  • "title" (optional): The title of the election.
  • "minSaved": The minimum number of preferences required for a ballot to be counted.
  • "minFormal": The minimum number of preferences required for a ballot to be counted as formal.
  • "groupsPerRow" (optional): The maximum number of group boxes in a row before wrapping. If absent, infinity.
  • "candidatesPerColumn" (optional): The maximum number of candidates in a column within a group box before wrapping. If absent, infinity.
  • "groups": A JSON object of the grouped candidates - use an empty object {} if there are no groups. The keys are the group identifiers. For each group, the value is an array of candidate identifiers.
  • "ungrouped": An array of ungrouped candidate identifiers - use an empty array [] if there are no ungrouped candidates.
Group and candidate identifiers must be unique and must not be an empty string "". A candidate identifier must not be the same as any group identifier. Group and candidate identifiers are also used as labels on the generated ballot guide.
